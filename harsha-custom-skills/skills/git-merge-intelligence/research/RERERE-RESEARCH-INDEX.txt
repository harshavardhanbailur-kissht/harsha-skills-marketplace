KEY TECHNICAL INSIGHTS:

✓ Content-based identification (not file-path based)
✓ Three-way merge verification ensures safety
✓ Imperceptible performance overhead (<100ms)
✓ Plain text storage for debugging transparency
✓ SHA-1 hash collision risk is negligible
✓ Requires manual finalization for safety
✓ Cannot handle: submodules, renames, binary files
✓ Works with: merge, rebase, cherry-pick, all merge strategies

DESIGN PATTERNS DOCUMENTED:

1. First conflict → resolve → record → future same conflict → auto-apply
2. Multi-target merge pattern (merge to multiple branches)
3. Long-lived branch pattern (keep branch synchronized)
4. Rebase-focused workflow (frequent rebases)
5. Feature branch integration (periodic syncs)

AI INTEGRATION OPPORTUNITIES:

1. Pre-rerere validation (confirm cached resolution before use)
2. Semantic enrichment (store intent metadata with resolutions)
3. Confidence scoring (rate cached resolution applicability)
4. Test-driven validation (run tests after auto-application)
5. Domain-aware routing (different resolution strategies per file type)
