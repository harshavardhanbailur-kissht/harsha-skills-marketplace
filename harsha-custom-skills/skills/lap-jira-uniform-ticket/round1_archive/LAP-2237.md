When a loan application is rejected, the system should automatically compute and set the customer's reapply date (cooling-off period) based on the rejection reason(s) tagged to the application.

**Logic:**

1. Maintain a master table mapping each rejection reason to its cooling-off period (in days), as per the table below.
2. At the time of rejection, read all rejection reasons applied to the application.
3. For each reason, look up the corresponding cooling-off period from the master.
4. **If the application has multiple rejection reasons, pick the maximum cooling-off period among them** and apply that as the reapply date (i.e., Reapply Date = Rejection Date + MAX(cooling-off days across all tagged reasons)).
5. If any reason maps to "Case-specific" (e.g., Age is Ineligible), flag the case for manual review instead of auto-setting the reapply date.
6. Cooling-off periods of `0` mean the customer can reapply immediately (reapply date = rejection date).
7. The reapply date should be stored against the customer/application and enforced at the point of new application creation - any attempt to reapply before this date should be blocked with a clear message showing the earliest reapply date.
8. The master table should be configurable (editable by business/ops without a code deploy) so cooling-off periods can be updated as policies change.

**Master Table - Rejection Reason to Cooling-Off Period:**

| Sr. No | Rejection Reason | Cooling-Off Period |
| --- | --- | --- |
| 1 | As requested by SM - wrong details entered | 0 |
| 2 | As requested by SM - need app / co-app in different application | 0 |
| 3 | PAN Verification Failed | 30 days |
| 4 | In 1–30 DPD - outstanding < ₹10K | 45 days |
| 5 | Manually Reject (standalone) | 0 |
| 6 | Low Bureau Score | 90 days |
| 7 | Bureau and banking not strong enough | 90 days |
| 8 | 30 DPD / Bureau 30 DPD | 90 days |
| 9 | Live loan overdue amount - CC | 60 days |
| 10 | Live loan overdue amount - Non CC | 60 days |
| 11 | In 1–30 DPD - outstanding ≥ ₹10K | 60 days |
| 12 | MFI Live Loan Overdue Check Failed | 90 days |
| 13 | Delinquency Check Failed | 90 days |
| 14 | 90 DPD Failed | 180 days |
| 15 | Write-off in last 12 months ≥ ₹10K | 180 days |
| 16 | Suit filed > 0 | 180 days |
| 17 | Age is Ineligible | 0 |
| 18 | Digi overdue | 30 days |
| 19 | Cabal count | 365 days |
| 20 | AA fraud | 365 days |

**Example:** If an application is rejected with "Low Bureau Score" (90 days) and "Suit filed > 0" (180 days), the reapply date = Rejection Date + 180 days.

Note - For any other reason not mentioned here but gets tagged as rejection reason the reapply date is set as 0.
