**Problem**

The issue tracker currently requires VPN to access. Off-roll employees (Product Support) cannot be given VPN access. Product Support works 6–7 days a week, often while traveling — they need to raise tickets from their phones whenever they have the details, not just from a laptop on VPN. Admins also need quick phone access to mark tickets as resolved on the go.

**What to do**

Host the web app on public internet. Google SSO already restricts access to @kissht.com domain — combined with the user whitelist (see LAP sibling ticket), this is sufficient access control for 5 active users.

**Why this is safe for now**

Only 5 people use this tool. Google SSO + domain restriction + whitelisted roles = adequate security. If user count grows significantly, revisit with IP allowlisting or a lightweight auth gateway.

**Acceptance Criteria**

- [ ] App accessible from any network (no VPN) on both desktop and mobile browsers
- [ ] Google SSO domain restriction remains enforced
