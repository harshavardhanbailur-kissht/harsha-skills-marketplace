### 1. Problem Statement

Currently in LSQ for Loan Against Property (LAP), PAN re-verification is not performed for existing users where the PAN would already have been verified during the Personal Loan (PL) journey or a previous LAP loan journey.

However, operational issues have been observed in some LAP cases where data mismatches exist (for example, mismatch in PAN-verified Date of Birth). Since LAP is a secured lending product, it is important to ensure that PAN details are verified again where required.

Therefore, a revised approach is required to ensure PAN validation while avoiding unnecessary re-verification where sufficient KYC validation already exists.

---

## 2. Proposed Solution

Whenever an **Existing-to-Business (ETB) user applies for a Loan Against Property (LAP)**, the system will first check the **VCIP completion status** and then determine whether PAN needs to be re-verified.

### Step 1: Check VCIP Status

For existing users entering the LAP flow, the system will check whether **VCIP has already been completed** **for any lender**.

### Step 2: PAN Verification Logic

**Case 1: VCIP Done = YES**
* PAN re-verification **will NOT be performed**.
* The existing verified PAN details will be considered valid.

**Case 2: VCIP Done = NO**
* PAN **will be re-verified** before allowing the user to proceed with the LAP application.

---

## 3. Applicability

The above PAN verification logic will apply to:

### Applicants
* Main Applicant
* Co-Applicant

### User Scenarios
* Existing Users (Existing in PL)
* Rejected Cases (Rejected Either in PL or even in LAP where VCIP is not done)
* Re-Login Cases (LAP Re-Login Cases)

### Loan Types
* Fresh LAP Loans
* BT + Top-Up Loans

---

## 4. Exception

PAN re-verification **will not be required** in the following scenario:

**Internal Top-Up Loan Cases**

For internal top-up applications, the system will **not trigger PAN re-verification**, irrespective of VCIP status.

---

## 5. Summary

| **Scenario** | **VCIP Status** | **Loan Type** | **PAN Re-verification** |
| --- | --- | --- | --- |
| ETB User applying for LAP | VCIP Done | Fresh / BT+TopUp | Not Required |
| ETB User applying for LAP | VCIP Not Done | Fresh / BT+TopUp | Required |
| ETB User applying for LAP | Any | Internal Top-Up | Not Required |
