Here are the flow related changes -

1. **Pre-initiation & Mandatory VCIP Check**

* **Existing KYC Check:** Before starting, LeadSquared (LSQ) must verify if the applicant has a successful, unexpired Video KYC (valid for 9 years and 6 months) from a previous SiCreva application (PL or LAP).
* **Bypass Protocol:** If a valid KYC is found, LSQ must bypass the KYC activity, mark it as "Completed," and change the case/applicant status to **"Onboarded"**. LSQ must then receive and store the masked Aadhaar and selfie from Kissht.
* **Mandatory Trigger:** If no valid KYC exists and **Aadhaar-PAN linkage is confirmed ("Yes")**, the system must make Video KYC mandatory and **disable Physical KYC** for that user.

2. **Digital Journey Initiation**

* **Link Generation:** LSQ triggers the journey, and Kissht must generate a unique link sent to both LSQ and the user via SMS. - Get the SMS verbiage verified
* **Digilocker Integration:**
    * If Digilocker was completed for SiCreva within the **last 72 hours**, the system must bypass the Digilocker stage and redirect the user directly to the VCIP journey.
    * If not completed within 72 hours, the user must complete the full Digilocker process and capture a selfie before proceeding to VCIP.
    * If a user drops off, the 72-hour Digilocker bypass remains active.

3. **VCIP Execution & Auditor Review**

* **Start Point:** The VCIP journey must begin with a **custom PAN card page**.
* **First Auditor Review:**
    * **Soft Reject:** The user must be allowed to redo the Video KYC using the same link for **72 hours**. If 72 hours pass, the user must redo both Digilocker and VCIP.
    * **Hard Reject:** A failure callback must be sent to LSQ, which will then **reject the application** and set "reapply days" to 0.
* **Second Auditor Review:**
    * **Approved:** A success screen must be shown to the user. Kissht must send a callback to LSQ with the **selfie, masked Aadhaar, and underwriting details**.
    * **Rejected:** A failure callback must be sent to LSQ, resulting in application rejection with "reapply days" set to 0.

**Note** - Also ensure for scenarios where we are either bypassing the KYC (either digilocker or VCIP), we should ask pass LSQ the selfie captured and the masked Aadhaar.
