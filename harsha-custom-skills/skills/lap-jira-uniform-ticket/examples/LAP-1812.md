**As a CPA user, I want to generate, manage consent, and complete signing of a combined document kit (KFS+SL, CAF, LA) for all eligible applicants post-sanction, so that the loan can proceed to disbursal with compliant documentation.**

---

## Primary Actor

* **CPA User (Internal / Assisted Journey)**

## Secondary Actors

* **Applicants & Co-Applicants (Non-Rejected Only)**
* **Digio (E-sign partner)**

---

## Preconditions

* Loan case stage is **Post Sanction**
* Loan is sanctioned successfully

---

## Document Composition Rules

* A **single combined PDF** must be generated in the following fixed order:
    1. **KFS + Sanction Letter (treated as one document)**
    2. **CAF**
    3. **Loan Agreement (LA)**
* All documents are **mandatory**
* Page references for **KFS & SL** must be dynamically picked via **system configuration** (varies by tenure due to repayment schedule)

---

## Functional Flow

### 1. Document Generation

* CPA user triggers document generation from LSQ at **Post Sanction stage**
* System generates a **single combined PDF**
* Any previous:
    * Signed documents
    * Unsigned documents
    * Consent and signing activity
      are **fully overwritten**, while **versioning of consent records is retained**

### 2. Physical Signing Flow

* Applicable when **Physical Sign** is selected
* CPA user uploads:
    * **One PDF only**
    * Combined signed document (KFS+SL + CAF + LA)
* Constraints:
    * File type: **PDF only**
    * Upload count: **Exactly one**
    * Uploaded file **cannot be replaced**
* Replacement is allowed **only if documents are regenerated**, which deletes prior documents

### 3. E-Signing Flow – Consent Capture

#### 3.1 Consent UI

* Hosted by **Kissht**
* Displays **only KFS + SL pages**
* Consent type: **Clickwrap**
* Consent is:
    * Captured **individually**
    * Mandatory for **all non-rejected applicants & co-applicants**
    * Captured via **unique SMS links** sent to each applicant

#### 3.2 Consent Storage

* Stored in:
    * **Kissht** → `user_consents` table
    * **LSQ** → Mortgage Details → System Generated Documents
* Track:
    * Consent status per applicant
    * Timestamp & metadata

### 4. Transition to Digio

* After consent, applicant is redirected to **Digio e-sign flow**
* Characteristics:
    * One Digio flow per applicant
    * Single OTP per applicant
    * All documents signed in one flow
    * Digio returns **one merged signed PDF**

### 5. Drop-off & Resume Logic

* If applicant drops:
    * **After consent but before signing** → resumes directly at Digio
    * **During Digio** → resumes from Digio OTP page
* Consent screen is **not shown again**
* Link validity:
    * **10 days**
    * Consent link expires **exactly with Digio link expiry**

### 6. Expiry Handling

* If link expires before signing:
    * Signing activity is deleted
    * CPA user can re-trigger document generation
    * Fresh links are sent to all applicants

### 7. Completion & Storage

* Once **all applicants**:
    * Have consented
    * Have signed successfully
* System stores:
    * Final **signed combined PDF**
    * Location: **Opportunity Data**
* Metadata captured:
    * Signed by (per applicant)
    * Consented by (per applicant)

### 8. Visibility & Tracking

* CPA user can view:
    * Consent status per applicant
    * Signing status per applicant
* Consent and signing statuses are displayed **separately**

## Acceptance Criteria

### Document Generation

* Combined PDF generated in correct order
* Regeneration invalidates old URLs and deletes documents
* Consent records are versioned

### Physical Signing

* Only one PDF upload allowed
* Replacement blocked unless regeneration happens

### Consent

* All non-rejected applicants must consent
* Consent captured individually via unique links
* Consent stored in both Kissht & LSQ

### E-Signing

* Digio triggered only after consent
* One OTP per applicant
* Single merged signed PDF returned

### Drop-off Handling

* Resume starts from Digio if consent already given
* Consent screen not repeated

### Expiry

* Links expire in 10 days
* Expired journeys require re-trigger

### Completion

* Signed document kit stored under Opportunity Data
* Signed-by and Consented-by metadata available
