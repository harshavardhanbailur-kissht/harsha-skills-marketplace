COMMON QUESTIONS ANSWERED
================================================================================

Q: Should I fine-tune embeddings?
A: Only if domain-specific retrieval performs poorly (<78% recall) and you 
   have 1000+ labeled pairs. Otherwise, use pre-trained models.

Q: What embedding dimension should I use?
A: Start with 384 (all-MiniLM). Only increase if accuracy insufficient and 
   latency not critical. Use matryoshka for flexibility.

Q: Is reranking worth the cost?
A: Only if accuracy > latency priority, substantial budget, or 
   retrieval baseline weak (<80% MRR).

Q: How often rebuild the index?
A: Static datasets: once. Dynamic: nightly full rebuilds or continuous updates.

Q: What if queries are short/ambiguous?
A: Use query expansion or HyDE (10-20% accuracy gains).

Q: How handle different languages?
A: Use multilingual embeddings (Cohere v4, Jina v3, BGE-M3).

Q: HNSW or IVF+PQ?
A: HNSW for most cases. IVF+PQ only if storage is critical constraint.

Q: Dense retrieval + ColBERT or Cross-encoder?
A: Dense + ColBERT: scale. Dense + Cross-encoder: precision.

Q: How much does production search cost?
A: MVP: $100/month. Growth: $1-3K/month. Enterprise: $3-10K/month.

Q: What's the HyDE improvement?
A: +65.9% vs baseline keyword search. Works best when retrieval poor.
